# kitcaf Blog

这是一个围绕 Notion 写作习惯构建的轻量博客方案。

## 背景

不知道大家是否存在这样的烦恼，每次发博客需要从笔记软件（notion）将内容复制到博客的后台，调样式。但是如果去构建专业的记笔记博客后台（感觉过于复杂化并且写博客体验不如专业的笔记软件），最重要的是/(ㄒoㄒ)/~~没有资源（服务器、域名、备案等太复杂了）

## 核心功能

- 用 Notion 写，而不是重新适应一套后台
- 只同步真正要发布的内容
- 生成静态博客，降低运行成本，可以部署到自己的轻量服务器，也可以放到支持静态站点的平台上
- 稳定处理 Notion 图片，Notion 自带图片链接不适合长期直接用于公开博客。同步时会把 Notion 图片缓存到稳定图床并构建缓存

## 推荐流程

```text
在 Notion 中写文章
  -> 整理标题、分类、标签和摘要
  -> 将状态改为 Published
  -> 触发部署脚本或自动化流水线
  -> 自动同步内容并构建静态博客
  -> 部署到服务器或静态托管平台
```

## 使用方式

项目的使用方式主要分为两种：自建服务器部署，或者使用 GitHub Actions 自动构建发布。两种方式的核心都一样：先把需要的变量配置完整，再交给脚本或自动化流程统一执行同步、构建和发布。

### 方式一：自建服务器

- 将项目代码放到服务器。
- 在服务器上配置 `.env`，填好 Notion、GitHub、图床和站点相关变量。
- 执行部署脚本，由脚本完成内容同步、静态构建和发布目录切换。
- 使用 crontab 或 systemd timer 定时执行脚本，让博客自动更新。

部署脚本入口：

```bash
scripts/deploy-self-hosted.sh
```

### 方式二：GitHub Actions

- 将代码托管到 GitHub。
- 将 Notion、GitHub、图床和站点相关变量配置到 GitHub Secrets。
- 在 Actions workflow 中调用项目的同步和构建流程。
- 通过定时触发或手动触发，让 GitHub Actions 自动生成并发布静态博客。

适合资源有限、希望降低维护成本的个人博客。

## Notion 字段约定

Notion 数据库建议包含以下字段：

- `Title`：文章标题
- `Status`：发布状态，只有 `Published` 会被同步
- `Category`：文章分类
- `Tags`：文章标签
- `PublishedAt`：发布时间
- `Slug`：文章访问路径
- `Description`：文章摘要

这些字段的目的不是增加写作负担，而是让文章在同步后可以自然生成列表、详情页、分类、标签和 SEO 信息。
