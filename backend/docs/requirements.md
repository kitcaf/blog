# 后端需求文档

## 项目概述

基于 Go 的博客系统后端服务，提供管理端和公开访问端的 RESTful API，支持 Notion 风格的块编辑器数据存储与同步。

## 核心功能需求

### 1. 认证与授权

- 管理员登录认证（JWT Token）
- Token 验证与刷新
- 登出功能
- 权限中间件保护管理端接口

### 2. 页面管理

- 支持树形结构的页面组织（父子关系）
- 页面元数据管理：标题、封面图、Slug、发布状态
- 页面的增删改查操作
- 草稿与发布状态控制
- 级联删除（删除页面时同步删除关联的所有 Block）

### 3. Block 内容管理

- 支持多种 Block 类型：段落、标题、列表、代码块、图片、Callout 等
- Block 的扁平化存储与检索
- 增量同步机制（UPSERT + DELETE）
- 事务保证数据一致性
- Block 与页面的关联关系

### 4. 公开访问接口

- 仅返回已发布的页面
- 页面导航菜单/目录树
- 单篇文章的 Block 数据获取
- 支持按 Slug 或 ID 访问

### 5. 性能优化

- Redis 缓存层（缓存已发布文章的 Block 数据）
- 缓存失效策略（内容更新时自动清除）
- 数据库查询优化
- 向量检索支持（为未来的语义搜索做准备）

## 非功能性需求

### 性能要求

- API 响应时间 < 100ms（无缓存）
- 缓存命中时响应时间 < 10ms
- 支持并发请求处理
- 增量同步接口支持批量操作

### 安全要求

- JWT Token 安全存储与传输
- SQL 注入防护
- XSS 防护
- CORS 跨域配置
- 敏感信息加密存储

### 可扩展性

- 支持水平扩展
- 数据库连接池管理
- 模块化架构设计
- 易于添加新的 Block 类型

### 可维护性

- 清晰的代码结构
- 完善的错误处理
- 日志记录
- API 文档

## 数据模型概要

### Pages 表
- 存储页面元数据
- 支持树形结构（parent_id）
- 包含发布状态、Slug、封面等字段

### Blocks 表
- 存储所有内容块
- 关联到具体页面（page_id）
- 包含类型、内容、顺序等字段
- 支持向量字段（为语义搜索预留）

### Users 表
- 管理员账户信息
- 密码哈希存储
- 角色权限字段

## 部署要求

- 支持 Docker 容器化部署
- 环境变量配置
- 数据库迁移脚本
- 健康检查接口
