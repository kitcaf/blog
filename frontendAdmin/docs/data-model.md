# 数据模型设计文档

## 1. Block 扁平化数据结构

### 1.1 核心概念

每篇文章的内容由一个 **Block 数组** 组成，每个 Block 是一个独立的内容单元。这种扁平化结构相比传统的树形结构有以下优势：

- **简单直观**: 数据结构清晰，易于理解和维护
- **易于存储**: 直接存储为 JSON，无需复杂的序列化
- **易于查询**: 可以对 Block 内容进行 JSON 查询
- **易于渲染**: 前端直接遍历数组渲染
- **易于扩展**: 添加新 Block 类型只需扩展数据结构

### 1.2 Block 基础结构

每个 Block 包含以下核心字段：

- **id**: 唯一标识符（UUID）
- **type**: Block 类型（段落、标题、列表、图片等）
- **props**: Block 属性（颜色、对齐、级别等）
- **content**: 内联内容（文本、链接、格式化）
- **children**: 子 Block 数组（用于嵌套结构）

### 1.3 支持的 Block 类型

#### 文本类 Block
- **paragraph**: 段落
- **heading**: 标题（支持 H1-H3）
- **quote**: 引用块
- **callout**: 提示框（信息、警告、错误、成功）

#### 列表类 Block
- **bulletListItem**: 无序列表项
- **numberedListItem**: 有序列表项
- **checkListItem**: 复选框列表项

#### 媒体类 Block
- **image**: 图片
- **video**: 视频（可选）
- **file**: 文件附件（可选）

#### 代码类 Block
- **code**: 代码块（支持语法高亮）

#### 其他 Block
- **divider**: 分割线
- **table**: 表格（可选）

### 1.4 Block 属性 (props)

不同类型的 Block 有不同的属性：

#### 通用属性
- **textColor**: 文字颜色
- **backgroundColor**: 背景颜色
- **textAlignment**: 文本对齐（左、中、右）

#### 标题特有属性
- **level**: 标题级别（1、2、3）

#### 代码块特有属性
- **language**: 编程语言（用于语法高亮）

#### 图片特有属性
- **url**: 图片 URL
- **caption**: 图片说明
- **width**: 图片宽度
- **alignment**: 图片对齐方式

#### 复选框特有属性
- **checked**: 是否选中

#### 提示框特有属性
- **type**: 提示类型（info、warning、error、success）

### 1.5 内联内容 (InlineContent)

每个 Block 的 content 字段是一个内联内容数组，支持：

#### 文本节点
- **text**: 纯文本内容
- **styles**: 文本样式
  - bold: 粗体
  - italic: 斜体
  - underline: 下划线
  - strikethrough: 删除线
  - code: 行内代码
  - textColor: 文字颜色
  - backgroundColor: 背景颜色

#### 链接节点
- **text**: 链接文本
- **href**: 链接地址
- **styles**: 链接样式（同文本样式）

### 1.6 嵌套结构

Block 支持嵌套，通过 children 字段实现：

- **列表嵌套**: 列表项可以包含子列表
- **引用嵌套**: 引用块可以包含其他 Block
- **提示框嵌套**: 提示框可以包含多个 Block

嵌套深度建议不超过 3 层，以保证性能和可读性。

## 2. PostgreSQL 数据库设计

### 2.1 articles 表

存储文章的基本信息和内容。

#### 字段说明

- **id**: UUID，主键
- **title**: 文章标题（VARCHAR 255）
- **slug**: URL 友好的标识符（VARCHAR 255，唯一）
- **content**: Block 数组（JSONB）
- **summary**: 文章摘要（TEXT，可自动提取或手动编辑）
- **category_id**: 分类 ID（UUID，外键）
- **published**: 是否发布（BOOLEAN，默认 false）
- **created_at**: 创建时间（TIMESTAMP）
- **updated_at**: 更新时间（TIMESTAMP）
- **deleted_at**: 删除时间（TIMESTAMP，软删除）

#### 索引设计

- **主键索引**: id
- **唯一索引**: slug
- **普通索引**: published, category_id, created_at
- **GIN 索引**: content（用于 JSONB 查询和全文搜索）

#### JSONB 存储优势

- **灵活性**: 无需预定义 Block 结构，易于扩展
- **查询能力**: 支持 JSON 路径查询、全文搜索
- **性能**: GIN 索引加速查询
- **原子性**: 整个文章内容作为一个事务更新

### 2.2 categories 表

存储文章分类，支持多级嵌套。

#### 字段说明

- **id**: UUID，主键
- **name**: 分类名称（VARCHAR 100）
- **slug**: URL 友好的标识符（VARCHAR 100，唯一）
- **parent_id**: 父分类 ID（UUID，外键，可为空）
- **description**: 分类描述（TEXT）
- **sort_order**: 排序顺序（INT，默认 0）
- **created_at**: 创建时间（TIMESTAMP）
- **updated_at**: 更新时间（TIMESTAMP）

#### 索引设计

- **主键索引**: id
- **唯一索引**: slug
- **普通索引**: parent_id, sort_order

#### 树形结构查询

使用 PostgreSQL 的递归 CTE (Common Table Expression) 查询分类树：

- **查询所有子分类**: 从根节点递归查询
- **查询父级路径**: 从子节点递归查询到根节点
- **查询同级分类**: 查询相同 parent_id 的分类

### 2.3 tags 表

存储文章标签。

#### 字段说明

- **id**: UUID，主键
- **name**: 标签名称（VARCHAR 50，唯一）
- **slug**: URL 友好的标识符（VARCHAR 50，唯一）
- **count**: 使用次数（INT，默认 0）
- **created_at**: 创建时间（TIMESTAMP）
- **updated_at**: 更新时间（TIMESTAMP）

#### 索引设计

- **主键索引**: id
- **唯一索引**: name, slug

### 2.4 article_tags 关联表

存储文章和标签的多对多关系。

#### 字段说明

- **article_id**: 文章 ID（UUID，外键）
- **tag_id**: 标签 ID（UUID，外键）
- **联合主键**: (article_id, tag_id)

#### 索引设计

- **联合主键索引**: (article_id, tag_id)
- **普通索引**: article_id, tag_id

#### 级联删除

- 删除文章时，自动删除关联记录
- 删除标签时，自动删除关联记录

### 2.5 files 表

存储上传的文件信息。

#### 字段说明

- **id**: UUID，主键
- **name**: 文件名（VARCHAR 255）
- **path**: 文件路径（VARCHAR 500）
- **url**: 访问 URL（VARCHAR 500）
- **size**: 文件大小（BIGINT，字节）
- **mime_type**: MIME 类型（VARCHAR 100）
- **created_at**: 创建时间（TIMESTAMP）

#### 索引设计

- **主键索引**: id
- **普通索引**: created_at

## 3. JSONB 查询能力

### 3.1 全文搜索

PostgreSQL 的 JSONB 支持全文搜索：

- **tsvector**: 将 JSONB 内容转换为全文搜索向量
- **tsquery**: 构建搜索查询
- **GIN 索引**: 加速全文搜索

### 3.2 JSON 路径查询

支持复杂的 JSON 路径查询：

- **提取特定类型的 Block**: 查询所有标题、图片等
- **搜索包含特定文本的 Block**: 模糊搜索
- **统计 Block 数量**: 统计文章长度
- **提取 Block 属性**: 提取图片 URL、代码语言等

### 3.3 查询示例场景

#### 搜索包含关键词的文章
在所有 Block 的文本内容中搜索关键词。

#### 提取所有图片
提取文章中所有图片的 URL 和说明。

#### 统计文章长度
统计文章包含多少个 Block。

#### 查询特定类型的 Block
查询文章中所有的标题或代码块。

## 4. 数据流设计

### 4.1 文章创建流程

1. **前端**: 用户在编辑器中创建内容
2. **前端**: 将 Block 数组序列化为 JSON
3. **前端**: 发送 POST 请求到后端 API
4. **后端**: 验证数据格式和权限
5. **后端**: 将 JSON 存储到 PostgreSQL JSONB 字段
6. **后端**: 返回创建的文章 ID
7. **前端**: 跳转到文章编辑页面

### 4.2 文章更新流程

1. **前端**: 用户编辑内容
2. **前端**: 防抖 2-3 秒后自动保存
3. **前端**: 发送 PUT 请求到后端 API
4. **后端**: 验证数据格式和权限
5. **后端**: 更新 JSONB 字段和 updated_at
6. **后端**: 返回更新结果
7. **前端**: 显示保存状态

### 4.3 文章查询流程

1. **前端**: 发送 GET 请求到后端 API
2. **后端**: 从 PostgreSQL 查询文章
3. **后端**: 返回 JSON 数据（包含 Block 数组）
4. **前端**: 解析 JSON 并渲染编辑器
5. **前端**: 用户可以继续编辑

### 4.4 文章发布流程

1. **前端**: 用户点击发布按钮
2. **前端**: 发送 POST 请求到发布 API
3. **后端**: 验证文章完整性（标题、内容、分类）
4. **后端**: 更新 published 字段为 true
5. **后端**: 返回发布结果
6. **前端**: 显示发布成功提示
7. **前端展示页**: 可以访问已发布的文章

## 5. 数据迁移和备份

### 5.1 数据迁移

#### 从 Markdown 迁移
- 解析 Markdown 文件
- 转换为 Block 数组
- 导入到数据库

#### 从其他编辑器迁移
- 解析原始数据格式
- 映射到 Block 结构
- 导入到数据库

### 5.2 数据备份

#### 定期备份
- 每日自动备份数据库
- 保留最近 30 天的备份
- 备份到云存储

#### 导出功能
- 导出为 JSON 格式
- 导出为 Markdown 格式
- 导出为 HTML 格式

## 6. 数据扩展性

### 6.1 添加新 Block 类型

只需要：
1. 定义新的 Block 类型名称
2. 定义新的 Block 属性
3. 前端实现渲染组件
4. 无需修改数据库结构

### 6.2 添加新的 Block 属性

只需要：
1. 在 props 中添加新字段
2. 前端实现属性编辑
3. 无需修改数据库结构

### 6.3 版本兼容性

- **向后兼容**: 新版本可以读取旧版本数据
- **优雅降级**: 不识别的 Block 类型显示为纯文本
- **数据迁移**: 提供迁移脚本升级旧数据

## 7. 性能优化

### 7.1 数据库优化

- **GIN 索引**: 加速 JSONB 查询
- **部分索引**: 只索引已发布的文章
- **连接池**: 复用数据库连接
- **查询优化**: 避免 N+1 查询

### 7.2 缓存策略

- **文章列表**: 缓存 5 分钟
- **文章详情**: 缓存 10 分钟
- **分类树**: 缓存 1 小时
- **标签列表**: 缓存 1 小时

### 7.3 数据压缩

- **JSONB 压缩**: PostgreSQL 自动压缩
- **Gzip 传输**: HTTP 响应 Gzip 压缩
- **图片压缩**: 上传时自动压缩

## 8. 总结

### 8.1 数据模型优势

1. **简单直观**: Block 数组结构清晰
2. **灵活扩展**: 易于添加新 Block 类型
3. **高效存储**: JSONB 原生支持
4. **强大查询**: JSON 路径查询和全文搜索
5. **易于维护**: 无需复杂的表关联

### 8.2 适用场景

- **博客系统**: 文章内容管理
- **知识库**: 文档管理
- **笔记应用**: 个人笔记
- **CMS 系统**: 内容管理

### 8.3 未来扩展

- **协同编辑**: 支持多人实时编辑
- **版本历史**: 记录每次修改
- **评论系统**: Block 级别的评论
- **权限控制**: Block 级别的权限
